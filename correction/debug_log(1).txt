Fig1 statistical method
-----------------------
Input:

- sbert_rq1_n50_baseline_by_item.csv
- sbert_rq1_n50_baseline_by_task.csv

Core formula:

- For each item, take the 5 original-prompt outputs.
- Compute all pairwise output similarities among the 5 outputs.
- There are 5 choose 2 = 10 pairs.
- mean_within_prompt_similarity = mean(pairwise similarities).
- sampling_noise_drift = 1 - mean_within_prompt_similarity.

The Fig1 plotting script then displays the item-level distribution of sampling_noise_drift by task, using violin/strip plots and task means.

Relevant script behavior:

- SimilarityModel.within_similarity() averages combinations(outputs, 2).
- calculate_baseline() computes avg_similarity and drift = 1 - avg_similarity.
- create_fig1()/figure_1() reads sbert_rq1_n50_baseline_by_item.csv and plots sampling_noise_drift.

Conclusion for Fig1:

The statistical method is consistent. Any visual differences between PNGs are rendering/layout/style differences, not formula differences.

Fig2 statistical method
-----------------------
Input:

- sbert_rq1_n50_baseline_by_task.csv
- rq1_n50_baseline_tukey.csv
- rq1_n50_baseline_significance_tests.csv

Core formula:

- Task mean = mean item-level sampling_noise_drift.
- Task std = sample standard deviation of item-level sampling_noise_drift.
- SE = std / sqrt(n_items).
- 95% CI = 1.96 * SE.
- Pairwise task comparison uses Tukey HSD on item-level sampling_noise_drift, alpha = 0.05.

The baseline significance file also contains:

- one-way ANOVA
- Kruskal-Wallis
- Levene test with median center

The Fig2 plotting script uses the task means, approximate 95% CIs, and Tukey compact letters.

Conclusion for Fig2:

The statistical method is consistent. The Llama-specific and unified plotting scripts both use the same task summary and Tukey CSV. Minor visual differences are due to plotting style and output filename differences, not statistical differences.

Fig3 statistical method
-----------------------
Input:

- sbert_rq1_n50_perturbation_effects_by_item.csv
- sbert_rq1_n50_perturbation_summary.csv
- sbert_rq1_n50_uncorrected_perturbation_summary.csv
- sbert_rq1_n50_heatmap_noise_corrected_drift.csv
- sbert_rq1_n50_uncorrected_heatmap_drift.csv

Core formula:

- baseline_similarity = within_similarity(original_outputs), same baseline as Fig1.
- perturbation_similarity = mean similarity across the Cartesian product of original outputs and perturbed outputs.
- With 5 original and 5 perturbed outputs, this is 25 cross pairs.
- uncorrected_drift = 1 - perturbation_similarity.
- noise_corrected_drift = baseline_similarity - perturbation_similarity.
- The heatmap cells are task x perturbation means over item-level drift values.

Relevant script behavior:

- SimilarityModel.cross_similarity() averages product(original_outputs, perturbed_outputs).
- build_analysis() computes baseline_similarity, perturbation_similarity, uncorrected_drift, and noise_corrected_drift.
- It writes both corrected and uncorrected heatmap CSVs.
- Fig3 plotting reads those two heatmap CSVs and renders before/after heatmaps.

Conclusion for Fig3:

The statistical method is consistent. The Fig3 PNG differences are due to presentation choices in the plotting scripts, not different drift calculations.
